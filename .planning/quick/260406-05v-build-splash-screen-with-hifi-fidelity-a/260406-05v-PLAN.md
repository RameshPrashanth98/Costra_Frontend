### Task 2: Rewrite SplashScreen.tsx with HIFI fidelity + animations

**Files:** `src/flows/splash/screens/SplashScreen.tsx`

**Action:** Replace the placeholder with a production component that:

1. **Layout:** Full-height (`min-h-[100dvh]`) flex column, page bg `#050505`, vertical sections distributed via gap/padding to match HIFI spacing.
2. **Eyebrow:** "COST OF LIVING TRACKER" in JetBrains Mono, lime, uppercase, tracked.
3. **Logo:** SVG — outer dark ring (`#0A0A0B` with subtle border `#1C1C1F`) containing a lime disc with the Costra "C" mark cut out (stroke style matches the image: blocky square "C").
4. **Wordmark:** Small lime "C" icon + "COSTRA" in Outfit 900, tracked.
5. **Title & body:** Per inventory table above; title uses `#FAFAFA`, body `#A1A1AA`.
6. **Loading section:** "Preparing your dashboard" mono label + 3 dots (active shifts every ~500ms in a 1→2→3 cycle, creating a pulse).
7. **Footer:** Footer tagline near bottom.
8. **Animations (CSS-only, no deps):**
   - On mount: elements fade+rise with staggered delays (eyebrow 0ms → logo 100ms → wordmark 250ms → title 400ms → body 550ms → loader 750ms → footer 900ms). Use CSS keyframes `splashFadeUp`.
   - Logo: subtle scale-in from 0.92 to 1 alongside fade.
   - Dots: sequential pulse (CSS `@keyframes splashDot` with staggered `animation-delay`) so the "active" dot appears to travel 1→2→3 continuously.
9. **Keep `<Link to="/welcome">` navigation** as an invisible click target over the logo OR as a tap-anywhere handler so the screen is still navigable in dev. Alternative: auto-navigate after 2.8s using `useEffect` + `setTimeout` + `useNavigate` — this matches "Preparing your dashboard" semantics. **Use auto-navigate** (with cleanup).
10. **Inline styles for all colors** per skill golden rules. Tailwind only for layout (flex, gap, padding, min-h-[100dvh], safe-area).
11. **Safe-area padding** on top and bottom using `pt-[max(env(safe-area-inset-top),48px)]` etc.
